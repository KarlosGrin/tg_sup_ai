# AI-ассистент для обработки данных в Telegram

Этот Telegram-бот, `@AI_my_sup_bot`, использует AI (Gemini/OpenAI) для анализа и обработки файлов, таких как Excel, CSV, Word и TXT. Пользователи могут загружать файлы и давать команды на естественном языке для выполнения различных операций с данными.

## 🚀 Возможности

- **Анализ файлов**: Автоматически определяет структуру Excel-листов, колонок CSV, абзацев и таблиц в Word.
- **Обработка данных**: Фильтрация, сортировка, группировка, удаление дубликатов, математические операции.
- **Трансформация**: Добавление вычисляемых колонок, объединение таблиц (аналог VLOOKUP), объединение листов.
- **Генерация отчетов**: Создание сводных таблиц, экспорт данных в разные форматы.
- **Конвертация форматов**: Преобразование файлов между `.xlsx`, `.csv`, `.docx`, `.txt`, `.xml` и другими.
- **Безопасное выполнение кода**: Задачи выполняются в изолированной "песочнице" для безопасности.

## 🛠️ Технологический стек

- **Бэкенд**: Python
- **Telegram Framework**: Aiogram 3.x
- **AI**: Gemini или OpenAI API
- **Обработка данных**: Pandas, NumPy
- **Работа с файлами**: openpyxl, python-docx, xlwt, odfpy

## ⚙️ Установка и запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <URL вашего репозитория>
    cd tg_sup_ai
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv venv
    source venv/bin/activate  # Для Windows: venv\Scripts\activate
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Настройте конфигурацию:**
    - Скопируйте файл `.env.example` в `.env`.
      ```bash
      cp .env.example .env
      ```
    - Откройте `.env` и вставьте ваши API-ключи и токен Telegram-бота.
    - **Никогда** не редактируйте `config_template.py` — он отслеживается git,
      и реальные ключи в нём утекут в публичный репозиторий.

5.  **Запустите бота:**
    ```bash
    python main.py
    ```

## 🐳 Docker-изоляция песочницы (рекомендуется для production)

### Модель безопасности (defense in depth)

| Слой | Технология | Что защищает |
|------|-----------|-------------|
| **1. Prompt injection** | Инструкция AI игнорировать команды из данных файла | Предотвращает генерацию опасного кода AI |
| **2. AST-анализ** | `ast.parse` + whitelist разрешённых dunder-атрибутов | Блокирует обход через конкатенацию строк, `getattr`, `__getattribute__` |
| **3. Блок-лист паттернов** | Substring search: `eval(`, `exec(`, `pickle`, `__subclasses__` и т.д. | Быстрый отсев очевидных атак |
| **4. Изоляция импортов** | `_safe_import` + whitelist разрешённых модулей | Нет `os`, `subprocess`, `socket`, `ctypes` |
| **5. Ограниченные builtins** | `__builtins__` без `eval`, `exec`, `compile`, `__import__` | Нельзя переопределить встроенные функции |
| **6. Безопасный `open()`** | Only в `downloads/<user_id>/` и `processed/<user_id>/` | Файловая система изолирована |
| **7. Docker-контейнер** | Полная изоляция ОС (см. ниже) | **Гарантированная граница безопасности** |

> ⚠️ **Важно:** Слои 1–6 — защита от случайных/наивных ошибок AI-генерации.
> **Единственная гарантированная security boundary — Docker-изоляция (слой 7).**
> Никогда не запускайте бота с `DOCKER_ENABLED=false` в продакшене.

Проект поддерживает два режима выполнения пользовательского кода:

### Режим 1: прямой subprocess (`DOCKER_ENABLED=false`)
- Код выполняется в отдельном Python-процессе на хосте
- Защита: AST-анализ + ограниченные builtins + блок-лист паттернов + whitelist импортов + таймаут (настраивается через `EXECUTION_TIMEOUT`, по умолчанию 120с)
- **⚠️ Не является полной изоляцией. Используйте ТОЛЬКО для разработки.**
- **Всегда включайте Docker для продакшена.**

### Режим 2: Docker-контейнер (`DOCKER_ENABLED=true` — **по умолчанию**)
- Код выполняется в изолированном контейнере с параметрами:
  - `--network none` — без доступа к сети
  - `--read-only` — файловая система только для чтения
  - `--memory=512m --cpus=1` — лимиты ресурсов
  - `--cap-drop ALL` — без привилегий
  - `--security-opt no-new-privileges:true` — запрет повышения прав
- **Настройка Docker:**
  1. Установите [Docker Desktop](https://www.docker.com/products/docker-desktop/)
  2. Соберите sandbox-образ:
     ```bash
     docker compose build sandbox
     ```
  3. В `.env` может потребоваться `DOCKER_ENABLED=true` (по умолчанию уже true)

### Запуск всего проекта в Docker (опционально)
```bash
docker compose up bot --build
```

## 📄 Конфигурация

Все настройки хранятся в файле `config.py`, который читает переменные из `.env` через `python-dotenv`.

| Переменная | Описание | По умолчанию |
|---|---|---|
| `BOT_TOKEN` | Токен Telegram-бота (от @BotFather) | — |
| `AI_PROVIDER` | Провайдер AI: `openai` или `gemini` | `openai` |
| `GEMINI_API_KEY` | Ключ Gemini API | — |
| `OPENAI_API_KEY` | Ключ OpenAI API | — |
| `ADMIN_IDS` | ID администраторов (через запятую) | — |
| `CHANNEL_ID` | ID канала Telegram (опционально) | — |
| `DOCKER_ENABLED` | Использовать Docker-изоляцию | `true` |
| `EXECUTION_TIMEOUT` | Таймаут выполнения кода (сек) | `120` |

Полный список — в файле `.env.example`.

## ⚠️ Важно

- Никогда не публикуйте файл `.env` и `config.py` — они содержат секретные ключи.
- Никогда не редактируйте `config_template.py` — он отслеживается git, и реальные ключи в нём утекут в публичный репозиторий.
- Для production-безопасности **обязательно включите `DOCKER_ENABLED=true`**.
