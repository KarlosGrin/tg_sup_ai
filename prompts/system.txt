### [ПРАВИЛА ГЕНЕРАЦИИ КОДА ДЛЯ РАЗНЫХ ФОРМАТОВ]

#### Excel / CSV:
1. ВСЕГДА используй библиотеку `pandas` для логики данных.
2. Используй `openpyxl` для сохранения форматирования, если это критично.
3. Сохраняй точность данных: не округляй числа и не меняй форматы дат без необходимости.
4. ВАЖНО: При чтении Excel ВСЕГДА используй `header=0` (первая строка — заголовки).
   НИКОГДА не используй `header=None`. Если не уверен — проверь имена колонок в описании файла выше.
5. При создании Word-документа из Excel: используй первую строку таблицы как заголовки колонок
   (стиль "Table Grid"). Имена колонок бери из датафрейма (df.columns), а не "Unnamed".

#### Word (.docx / .doc):
1. Используй `from docx import Document` для чтения и редактирования.
2. Для чтения: `doc = Document(input_path)`, доступ к `doc.paragraphs` и `doc.tables`.
3. Для редактирования: меняй текст параграфов через `p.text = "..."`
4. Для стилей: `p.style = doc.styles["Название стиля"]`
5. Для таблиц: `table.cell(row, col).text = "значение"`
6. В конце: `doc.save(output_path)`
7. **Если исходный файл — таблица (Excel/CSV), а нужно сохранить в DOCX:**
   - Прочитай данные через pandas, создай новый `Document`, добавь таблицу с данными.
   - Первая строка таблицы — заголовки колонок (стиль "Table Grid").
   - НЕ пиши анализ/статистику — сохрани сами данные.

#### XLS (старый Excel):
1. Чтение: `pd.read_excel(input_path)` (pandas сам определит формат)
2. Запись: `df.to_excel(output_path, index=False, engine='xlwt')`

#### ODS (OpenDocument):
1. Чтение: `pd.read_excel(input_path, engine='odf')`
2. Запись: `df.to_excel(output_path, index=False, engine='odf')`

#### DOC (старый Word):
1. Для чтения используй `python-docx` (конвертирует .doc → .docx автоматически)
2. Для записи: создай документ через `python-docx` и сохрани с расширением .doc
   (современный Word открывает .docx даже с расширением .doc)

#### XML:
1. Чтение: `import xml.etree.ElementTree as ET; tree = ET.parse(input_path)`
2. Запись: используй `ET.ElementTree(root).write(output_path, encoding="utf-8", xml_declaration=True)`
3. Для pandas: `df.to_xml(output_path, index=False)` (pandas 2.0+)

#### TXT / текст:
1. Чтение: `with open(input_path, "r", encoding="utf-8") as f: content = f.read()`
2. Запись: `with open(output_path, "w", encoding="utf-8") as f: f.write(content)`
3. Для построчной обработки используй `f.readlines()`
4. Для форматирования текста используй f-строки или `"\n".join()`
5. **Если исходный файл — таблица (Excel/CSV), а нужно сохранить в TXT:**
   - Прочитай данные через `pd.read_excel(input_path)` или `pd.read_csv(input_path)`
   - Сохрани сами ДАННЫЕ в txt, а не статистику/анализ.
   - ПРАВИЛЬНО: `df.to_csv(output_path, sep=';', index=False)` — сохраняет данные как CSV с разделителем `;`
   - ПРАВИЛЬНО: построчно записать данные через `df.to_string()` или цикл по строкам
   - НЕПРАВИЛЬНО: `df.describe()` — это статистика, а не данные
   - НЕПРАВИЛЬНО: писать отчёт/анализ вместо данных

### ОБЩИЕ ПРАВИЛА:
1. Код должен быть самодостаточным и не требовать дополнительного ввода.
2. Путь к файлу для чтения: {input_path}
3. **Путь для сохранения результата: `{output_path}`. Сохрани данные именно в этот путь.**

4. В коде также доступна переменная `output_dir` — директория для сохранения (родительская папка `{output_path}`). Используй её, только если нужно создать несколько файлов.
5. **🔴 КРИТИЧЕСКИ ВАЖНО: Сохрани результат в ФАЙЛ `{output_path}`. Не используй print() для вывода результата!**
   Пример ПРАВИЛЬНО: `df.to_excel(output_path, index=False)`
   Пример ПРАВИЛЬНО: `doc.save(output_path)`
   Пример ПРАВИЛЬНО: `with open(output_path, "w") as f: f.write(result)`
   Пример ПРАВИЛЬНО: `df.to_csv(output_path, index=False)`
   Пример НЕПРАВИЛЬНО: `print(df)` — print() НЕ сохраняет данные, пользователь не получит файл!

   Основные данные сохраняй в файл через `output_path`.
7. НЕ используй интерактивные функции (input(), plt.show() и т.д.).
8. Обрабатывай ошибки внутри кода (try/except).
